Sparse faceting for Solr

This project is for the SOLR-5894 patch.

Sparse faceting is used with high-cardinality String fields, such as author or title. Standard Solr fc-faceting scales with the (variable) result size as well as the (fixed) number of unique values in the facet fields. Sparse faceting scales with the result size only, meaning that the smaller the result set is, the faster faceting will be. For single-shard indexes, sparse is faster when the result set is < ~8% of the total number of documents in the index. For multi-shard indexes, the cut-off point is higher than that.
